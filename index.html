1. 사업 개요

핵심 가치: 정신의학 × 개인화 동화책. 재미, 감동, 부모-아이 상호작용 개선을 동시에 달성하는 세계 유일의 제품.

제품: 아이 사진을 AI가 분석하여 동화책 장면에 합성하는 개인화 동화책

창업 철학: 현금흐름 우선

이전에 총 투자금 약 8억원을 소진하고, 직원 11명에 월 인건비 5,000만원 이상을 운영한 경험이 있다. 그 경험에서 배운 것은 명확하다: 사람을 먼저 뽑고 매출을 만드는 구조는 틀렸다. 매출이 먼저고, 사람은 매출이 정당화할 때 뽑는다.

이번에는 다르게 간다.

  1. 1인 + AI 자동화로 시작한다. 제품 설계(정신의학), 기술 개발(AI 파이프라인), 운영 자동화를 대표 혼자 + AI 도구로 해결한다. 2026년 현재 AI 코딩 어시스턴트, AI 이미지 생성, AI 고객 응대가 실무 수준에 도달했다. 11명이 하던 일을 1명 + AI가 할 수 있는 시대다.

  2. 첫 달부터 흑자를 만든다. 고정비를 극단적으로 낮추면 월 225권만 팔아도 손익분기다. 1,000권이면 월 2,000만원 이상의 영업이익이 나온다. J커브를 그리며 적자를 감내하는 게 아니라, 첫 달부터 현금이 쌓이는 구조.

  3. 채용은 매출이 증명한 후에 한다. 월 3,000권을 넘기고, 물리적으로 혼자 처리할 수 없는 병목이 생겼을 때 첫 번째 직원을 뽑는다. 그 사람의 급여는 이미 만들어진 현금흐름에서 나온다.

  4. 투자는 필요 없다. 선택이다. 현재 수억원의 현금을 보유하고 있어 런칭 자금은 충분하다. 50~100억원 이하의 투자는 받을 이유가 없다. 투자를 받는다면, 이미 검증된 유닛 이코노믹스를 폭발적으로 스케일하기 위해서만.

왜 이 제품은 나만 만들 수 있는가

2019년부터 지금까지 7년. 수천 권의 동화책을 직접 만들었다. 고객에게 사진을 받고, 배경을 제거하고, 장면에 배치하고, 텍스트를 넣고, 인쇄 파일을 만들어 인쇄소에 넘기는 전 과정을 손으로 했다. 수천 장의 아이 사진을 직접 다뤄본 사람만이 아는 것들이 있다 — 어떤 사진이 인쇄에서 깨지는지, 어떤 포즈가 장면에 자연스럽게 들어가는지, 부모가 어떤 순간에 감동하고 어떤 순간에 실망하는지.

그리고 그 제품을 설계한 것도 나다. 소아정신의학을 공부한 사람이 동화책의 매 페이지를 설계했다. 아이가 자기 이름을 발견하는 순간의 놀라움, 부모가 읽어줄 때 자연스럽게 일어나는 대화, 반복해서 읽을수록 아이에게 스며드는 메시지 — 이것은 마케팅 카피가 아니라 설계 원리다.

그리고 그것을 자동화하는 코드를 짠 것도 나다. AI 배경 제거 모델 6종을 직접 서빙하고, 포즈 분석 파이프라인을 만들고, 품질 검수 시스템을 만들고, 모바일 미리보기 앱을 만들었다.

이 세 가지 — 수천 권의 수동 제작 경험, 정신의학 기반 제품 설계, 프로그래밍 — 를 한 사람이 가지고 있다는 것이 핵심이다.

왜 이것이 중요한가. 개인화 동화책에는 "적정 영역"이 존재한다.

자동화의 선을 너무 높이 잡으면 — 예를 들어 AI가 아이 외모를 기반으로 캐릭터를 실시간 생성하는 수준 — 현재 기술로는 인쇄 품질을 보장할 수 없고, 부자연스러운 결과물이 나와 클레임이 된다. 반대로 자동화의 선을 너무 낮게 잡으면 — 이름만 바꿔 넣는 수준 — 제품이 유치해지고, 88,000원은커녕 15,000원도 받기 어렵다.

자동화가 가능하면서, 유저에게 "세상에 하나뿐인 내 아이의 책"이라는 감동을 주면서, 정신의학적으로도 의미 있는 제품. 이 세 조건을 동시에 만족하는 적정 영역을 찾고, 그 영역 안에서 극한까지 끌고 갈 수 있는 사람은 이 지구상에 나뿐이다. 수동 제작을 해보지 않은 기술자는 인쇄 품질의 함정을 모르고, 코드를 모르는 기획자는 자동화의 한계를 모르고, 정신의학을 모르는 사람은 "감동"과 "유치함"의 경계를 모른다.

7년간 실행하고, 구상하고, 좌절하고, 회복했다. 8억을 쓰고 11명을 뽑아봤고, 그게 틀렸다는 것도 배웠다. 이번에는 혼자, AI와 함께, 제대로 한다.

3. 채용 — Job Descriptions

아래 JD는 Phase 1~2에서 매출이 채용을 정당화할 때 사용한다. 지금 당장의 채용 계획이 아니라, 각 포지션이 필요해지는 시점에 즉시 실행할 수 있도록 미리 설계해둔 것이다.

우리는 정신의학을 개인화 동화책에 녹여, 부모와 아이 사이에 진짜 대화가 일어나게 만드는 회사입니다. 권당 8만 8천원에도 팔렸던 제품입니다. 기술로 가격을 절반 이하로 낮추고, 월 수만 명의 가정에 도달하려 합니다.

1. COO / 비즈니스 매니저 (Phase 2 — 대표가 제품에 집중 못할 때)

이 포지션이 존재하는 이유

우리 대표는 소아정신의학 관점에서 동화책을 설계하는 사람입니다. 이 사람이 재무제표를 보고, 세무사와 통화하고, 인쇄사 단가를 협상하는 시간은 회사에 손해입니다. 당신이 오면 대표는 제품 개발에만 집중하게 됩니다. 나머지 모든 것은 당신이 가져갑니다.

당신이 할 일

당신은 제품 개발을 제외한 회사의 모든 운영을 총괄합니다.

재무와 숫자. 월 매출 5억 이상의 현금흐름을 관리합니다. 원가 구조를 꿰뚫고 있어야 합니다. 인쇄 단가 20,000원, 포장배송 3,000원, PG수수료 3%, 광고비 비중 — 이 숫자들이 1%씩 움직일 때 영업이익에 어떤 영향을 주는지 즉시 판단할 수 있어야 합니다. 세무회계는 외부 전문가를 쓰되, 그들에게 정확한 질문을 던지는 것은 당신의 몫입니다.

파트너십과 협상. 인쇄사(북토리)와의 관계가 핵심입니다. 월 1만권이 3만권이 되었을 때 단가를 어떻게 가져갈지, 납기가 밀릴 때 백업 인쇄사는 어디인지, 이런 판단을 내려야 합니다. Cafe24, 아임포트, 알리고, SweetTracker 등 외부 서비스와의 계약 관리도 당신의 영역입니다.

B2B 영업. 어린이집 졸업앨범, 소아과 대기실 비치, 기업 복지몰 입점 — 이런 채널을 직접 개척합니다. 권당 8만 8천원에도 개인 고객이 구매한 제품입니다. B2B에서는 볼륨 할인을 줘도 충분한 마진이 남습니다. 이 채널이 전체 매출의 20~30%가 되도록 만드는 것이 1년 내 목표입니다.

투자 대응. 우리는 소액 투자를 구하지 않습니다. 하지만 유닛 이코노믹스가 검증된 후 50~100억 규모의 스케일업 투자를 받을 때, IR 자료를 만들고 초기 미팅을 진행하는 것은 당신의 몫입니다. 대표는 최종 미팅에만 참석합니다.

조직. 5명 이하의 극도로 작은 조직이지만, 각자가 자기 영역에서 독립적으로 움직여야 합니다. 채용, 온보딩, 성과 관리의 기본 체계를 만듭니다. 대표에게 "이 사람을 뽑아야 합니다"라고 제안하는 것도, "이 프로세스가 병목입니다"라고 보고하는 것도 당신의 역할입니다.

방패. 대표에게 올라오는 모든 잡무, 긴급하지만 중요하지 않은 일, 외부의 불필요한 미팅 요청을 차단합니다. 대표의 캘린더에 "제품 개발"만 남아있게 만드는 것이 당신의 가장 중요한 KPI입니다.

당신에게 필요한 것

  • 스타트업 또는 커머스 비즈니스에서 운영/경영기획 경험 3년 이상
  • P&L을 직접 관리해본 경험. 매출이 아니라 영업이익으로 사고하는 습관
  • 협상 경험. 공급사, 유통사, 또는 투자자와 테이블에 앉아본 적이 있을 것
  • 스프레드시트와 숫자에 강할 것. 대시보드를 직접 만들 수 있으면 더 좋음
  • "대표가 신경 쓰지 않아도 되게 만드는 것"에서 보람을 느끼는 성향
  • 완벽한 시스템보다 빠른 실행을 선호할 것. 당신이 합류할 때 회사는 이미 월 수천만원의 이익을 내고 있습니다

필요하지 않은 것

  • 대기업 전략기획 경험 (우리에겐 실행이 필요합니다)
  • 동화책/출판 업계 경험 (있으면 좋지만, 없어도 됩니다)

2. CTO (Phase 2 — 기술 부채 해소 + R&D 동시 필요 시)

이 포지션이 존재하는 이유

고객이 아이 사진을 업로드하면, AI가 배경을 제거하고, 포즈를 분석하고, 20페이지 동화책에 자동으로 배치하고, 부모에게 미리보기를 보여주고, 승인되면 인쇄 파일을 생성하여 인쇄사에 자동 발주합니다. 이 전체 파이프라인이 현재 작동하고 있습니다. 당신은 이것을 인수받아 월 1만건, 그 다음 5만건까지 스케일하는 사람입니다.

당신이 할 일

AI 파이프라인. 현재 시스템은 Python FastAPI 서버에서 배경 제거(ben2, hr-matting)와 포즈 감지(BlazePose, face-api.js)를 수행합니다. RTX 4070 Super 1장으로 돌아가고 있습니다. 월 1만건은 GPU 2~4장이면 충분하지만, 3만건을 넘어가면 아키텍처를 다시 생각해야 합니다. 로컬 GPU와 클라우드 GPU를 어떻게 조합할지, 배치 처리와 실시간 처리를 어떻게 분리할지, 이런 결정을 내립니다.

프론트엔드. 부모가 사용하는 미리보기 앱(book-preview)은 Vanilla JS로 작성되어 있습니다. 모바일 사파리에서 매끄럽게 동작해야 합니다. HEIC 변환, 터치 드래그, 캐러셀 애니메이션, 폴라로이드 앨범 레이아웃 등 이미 복잡한 기능이 들어가 있습니다. 첫 번째 테마(NAME)는 config 기반으로 작동하고 있지만, 모든 테마를 config만으로 생성할 수 있는 범용 구조는 아직 완성되지 않았습니다. 당신은 이 구조를 범용화하고 고도화합니다.

자동화. 현재 수동인 단계를 없앱니다. Cafe24에서 주문이 들어오면 → 고객에게 사진 업로드 알림톡 발송 → 사진 수신 → AI 처리 → 미리보기 생성 → 고객 승인 → 인쇄 파일 생성 → 북토리 API 자동 발주 → 배송 추적 연동. 이 체인에서 사람의 개입이 0에 가까워야 합니다.

품질 자동 검수. 인쇄 사고는 절대 일어나면 안 됩니다. 얼굴이 잘리거나, 해상도가 부족하거나, 이상한 사진이 동화책에 들어가면 클레임이 됩니다. 현재 pre-flight check 시스템이 있지만, 정확도를 99% 이상으로 올려서 수동 검수를 거의 제거하는 것이 목표입니다.

인프라와 보안. 아이 사진은 민감한 개인정보입니다. 처리 후 즉시 삭제되는 파이프라인, 전송 구간 암호화, 접근 로그 관리가 필요합니다. 현재 Cloudflare Tunnel로 서버를 노출하고 있는 구조도 장기적으로는 재설계가 필요합니다.

대표와의 협업. 대표가 새로운 동화책 테마를 설계하면, 당신은 그것을 config 파일 하나로 시스템에 올릴 수 있는 구조를 만들어야 합니다. "새 스토리 20페이지, 일러스트 20장, 텍스트 A/B 버전" — 이것만 넘기면 바로 프로덕션에 올라가야 합니다.

당신에게 필요한 것

  • Python 백엔드 실무 경험 3년 이상 (FastAPI/Flask/Django)
  • 프론트엔드 개발 능력 (JS/HTML/CSS — React 없이도 복잡한 UI를 만들 수 있을 것)
  • GPU 서버 운영 경험 (CUDA, 모델 서빙, 배치 처리)
  • AI/ML 모델을 직접 학습시키지는 않아도 되지만, 오픈소스 모델을 평가하고 서빙하고 파인튜닝 지시를 내릴 수 있을 것
  • API 설계와 시스템 통합 경험 (외부 서비스 5개 이상이 엮인 구조를 다뤄본 적 있을 것)
  • "혼자 다 하는" 것에 익숙한 풀스택. 우리에게 지금 필요한 건 스페셜리스트가 아니라 제너럴리스트입니다
  • 장애 대응 — 새벽에 서버가 죽으면 당신이 일어나야 합니다. 이것이 불편하면 이 자리는 맞지 않습니다

필요하지 않은 것

  • 대규모 팀 매니지먼트 경험 (처음엔 혼자, 나중에 1~2명 리딩)
  • 논문 쓸 수준의 ML 역량 (우리는 모델을 만드는 회사가 아니라 사용하는 회사입니다)

3. 운영/CS 매니저 (Phase 1 — 첫 번째 채용, 월 3,000권 초과 시)

이 포지션이 존재하는 이유

기술이 아무리 좋아도, 고객이 "사진을 잘못 올렸어요", "아이 이름 철자가 틀렸어요", "언제 배송되나요"라고 물었을 때 빠르고 따뜻하게 응대하지 못하면 브랜드가 망가집니다. 아이를 위한 선물을 주문한 부모는 일반 쇼핑몰 고객보다 감정적 기대치가 높습니다. 당신은 이 기대를 충족시키는 동시에, 하루 100~300건의 주문이 막힘 없이 흘러가게 만드는 사람입니다.

당신이 할 일

일일 생산 관리. 매일 아침 대시보드를 열어 전날 주문 상태를 확인합니다. 사진 미수신, AI 처리 실패, 고객 미승인, 인쇄 지연 — 각 단계에서 멈춰있는 건을 찾아내고 해결합니다. D+3 출고라는 SLA를 지키는 것이 당신의 핵심 지표입니다. 단 1건이라도 7일 넘게 지연되면, 왜 그런지 원인을 파악하고 재발을 막아야 합니다.

품질 검수. AI가 자동으로 처리한 동화책 중 약 5~10%는 "사람이 한번 봐야 할 것 같다"는 플래그가 뜹니다. 아이 얼굴이 가려진 사진, 해상도가 애매한 사진, 두 명이 찍힌 사진에서 주인공을 잘못 잡은 경우 등. 하루 25~50건 정도를 빠르게 판단하여 통과시키거나, 고객에게 사진 재업로드를 요청합니다. 이 판단이 느리면 전체 파이프라인이 밀립니다.

고객 응대. 카카오톡 채널, Cafe24 게시판, 전화로 들어오는 문의에 대응합니다. AI 챗봇이 배송 조회, FAQ 같은 단순 문의를 1차로 처리하지만, "사진을 바꾸고 싶어요", "이름 한자가 틀렸어요", "선물 포장 가능한가요" 같은 문의는 사람이 해야 합니다. 일 50~100건 정도의 에스컬레이션을 예상합니다. 핵심은 속도와 톤입니다. 부모는 아이를 위한 특별한 선물을 기다리고 있다는 것을 항상 기억해야 합니다.

클레임과 재제작. 인쇄 불량, 오탈자, 사진 품질 불만 등으로 재제작이 필요한 경우가 있습니다. 전체의 1~2% 정도지만, 이 고객들의 경험이 가장 중요합니다. 무조건 재제작을 해주되, 원인을 기록하여 대표에게 피드백합니다. 같은 원인의 클레임이 반복되면 시스템을 고쳐야 하니까요.

VOC 수집. 고객이 "이런 테마도 있으면 좋겠어요", "읽어주기 가이드가 너무 좋았어요", "글자가 좀 작은 것 같아요"라고 말할 때, 이것을 체계적으로 기록하여 주간 리포트로 만듭니다. 대표가 다음 제품을 설계할 때 가장 먼저 보는 자료가 이것입니다.

당신에게 필요한 것

  • 커머스 또는 서비스 운영/CS 경험 2년 이상
  • 하루 50건 이상의 문의를 처리해본 경험. 멀티태스킹에 강할 것
  • 엑셀/시트로 데이터를 정리하고 패턴을 찾아낼 수 있을 것
  • 감정적인 고객에게도 차분하고 따뜻하게 대응할 수 있는 사람
  • "이 문제가 왜 생겼지?"를 파고드는 성향. 같은 문제가 두 번 일어나는 것을 참지 못하는 사람
  • AI 도구(챗봇, 자동화)를 두려워하지 않고 적극적으로 활용하려는 태도

필요하지 않은 것

  • 콜센터 대규모 운영 경험 (우리는 소규모 고품질 CS입니다)
  • 개발 능력 (하지만 어드민 대시보드를 능숙하게 사용할 수 있어야 합니다)

4. 마케팅 매니저 (Phase 2 — 월 광고비 3,000만원 이상 시)

이 포지션이 존재하는 이유

이 제품은 권당 8만 8천원에도 팔렸습니다. 아이 사진과 이름이 들어간 세상에 하나뿐인 동화책이라는 가치를 부모가 이해하는 순간, 구매 전환은 높습니다. 문제는 그 순간을 만드는 것입니다. 당신은 "정신의학 기반 개인화 동화책"이라는, 시장에 없던 카테고리를 부모들의 머릿속에 만드는 사람입니다.

당신이 할 일

포지셔닝 설계. 우리는 "이름이 들어간 동화책"이 아닙니다. "아동 심리 전문가가 설계한, 부모-아이 상호작용을 만드는 동화책"입니다. 이 차이를 3초 안에 전달할 수 있는 메시지를 만들어야 합니다. 기존 커스텀 동화책 시장과 우리를 명확하게 구분하는 브랜딩이 필요합니다.

퍼포먼스 마케팅. 메타(인스타그램/페이스북), 네이버 검색광고, 카카오모먼트를 집행합니다. 월 광고비는 매출의 10~15% 수준에서 시작하여, ROAS를 보며 조정합니다. 당신이 합류하는 시점에는 대표가 이미 ROAS가 검증된 채널을 만들어놓은 상태입니다. 0~4세 자녀를 둔 부모, 돌잔치 준비 중인 부모, 출산 선물을 찾는 사람 — 타겟 세그먼트별로 다른 소재와 메시지를 운영합니다. A/B 테스트를 멈추지 않습니다.

콘텐츠 마케팅. 맘카페(맘스홀릭, 레몬테라스 등)에서 자연스러운 체험 후기가 퍼지게 만드는 것이 가장 강력한 채널입니다. 체험단을 월 50~100명 규모로 운영하고, 진짜 후기를 받아 2차 콘텐츠로 가공합니다. 인스타그램에서는 "아이가 자기 이름이 나오는 책을 보고 놀라는 영상"이 바이럴 소재입니다. 이런 순간을 포착하는 감각이 있어야 합니다.

전문가 마케팅. 대표가 설계한 정신의학적 근거를 마케팅 자산으로 전환합니다. 소아정신과 전문의의 추천사, 아동심리상담사의 리뷰, 육아 인플루언서와의 협업 — "전문가가 인정한 동화책"이라는 신뢰를 쌓습니다. 이것은 단순 체험단과는 다른 차원의 마케팅입니다.

CRM. 한 번 구매한 고객이 다시 사게 만듭니다. 아이는 자라니까요. 돌에 "이름 동화"를 산 부모에게 3세에 "감정 동화"를, 5세에 "관계 동화"를 제안합니다. 출산 예정일, 아이 생일 기반 자동 리마인드를 설계합니다. 형제가 있는 가정에는 "둘째 동화책"을 추천합니다.

데이터. CAC(고객획득비용), LTV(고객생애가치), 전환율, 재구매율, ROAS — 이 숫자들을 매일 봅니다. "감으로 하는 마케팅"은 우리에게 없습니다. 어떤 채널에서, 어떤 메시지로, 어떤 타겟에게 접근했을 때 전환이 일어나는지를 데이터로 증명하고, 그 채널에 돈을 집중합니다.

당신에게 필요한 것

  • 퍼포먼스 마케팅 실무 경험 3년 이상 (메타, 네이버 필수)
  • 월 3,000만원 이상의 광고비를 직접 집행하고 ROAS를 관리해본 경험
  • 육아/키즈 시장에 대한 이해 또는 관심. 0~8세 자녀를 둔 부모의 구매 심리를 아는 사람
  • 콘텐츠 감각. 카피 한 줄, 썸네일 한 장이 전환율을 바꾼다는 것을 경험으로 아는 사람
  • GA4, 메타 픽셀, 네이버 전환 추적 등 데이터 트래킹 설정을 직접 할 수 있을 것
  • AI 도구를 활용한 콘텐츠 제작에 거리낌이 없을 것 (카피라이팅, 이미지, 영상 편집)

필요하지 않은 것

  • 브랜드 마케팅만 해본 사람 (우리는 퍼포먼스와 브랜딩을 동시에 해야 합니다)
  • 대행사 경험만 있는 사람 (인하우스에서 P&L 책임을 져본 경험이 중요합니다)

대규모 스케일 시 추가 가능 포지션

아래 포지션은 월 30,000권 이상, 해외 진출 등 현재 계획의 범위를 넘어서는 스케일에서만 고려한다. 당분간 필요 없음.

5. 풀스택 개발자

CTO의 백업이자 실행 파트너. 매일의 기능 개발과 버그 수정을 담당. Python + JavaScript 풀스택이어야 하며, "혼자서 기능 하나를 처음부터 끝까지 배포할 수 있는" 수준이 필요.

6. 크리에이티브 프로듀서

B2B 영업과 해외 파트너십 전담. 어린이집, 소아과, 기업 복지몰, 해외 현지 인쇄/물류.

5.6 투자에 대한 입장

투자가 필요한가? 아니다. 현금 수억원 보유, 월 225권만 팔면 흑자, Year 1 영업이익만 7억원. 생존을 위한 투자는 필요 없다.

그렇다면 왜 투자를 고려하는가?

이미 작동하는 엔진에 기름을 부어 폭발적으로 키우기 위해서다. 구체적으로:

투자금 용도 금액 규모 효과
공격적 마케팅 월 5~10억 ROAS가 검증된 채널에 자금 집중, 월 5만건+ 돌파
해외 진출 10~20억 일본/미국 현지 인쇄, 물류, 마케팅
인쇄 내재화 10~15억 인쇄 원가 20,000원 → 10,000원 이하, 마진 극대화
팀 빌딩 10~15억 CTO + 핵심 개발자, 글로벌 운영팀

50~100억원 이하의 투자는 받지 않는다. 소액 투자를 받아 IR에 시간을 쓰는 것은 제품 개발과 현금 창출에 방해가 된다. 투자를 받는 시점은 월 매출 5억 이상, 유닛 이코노믹스가 완전히 검증된 이후. 그때 50~100억원을 받아 글로벌 1위 개인화 동화책 회사로 만든다.

인쇄 사고를 사전에 차단하는 AI 검수 시스템. 8개 항목을 자동 체크한다:

검수 항목 내용
얼굴 영역 검증 얼굴이 잘리거나 너무 작지 않은지
마스크 무결성 배경 제거가 깨끗한지
구멍 감지 전경에 뚫린 부분이 없는지

전경 일관성 색상/조명이 자연스러운지
소지품 검증 들고 있는 물건이 잘리지 않았는지
잔여물 감지 배경 찌꺼기가 남아있지 않은지

아동(KID_PERSON), 성인(ADULT_PERSON), 사물(TOY_OBJECT) 등 케이스별로 다른 기준을 적용한다. 0~100점으로 점수화하여, 기준 미달 시 자동 플래그.

④ Config-Driven 동화책 시스템

새로운 테마를 추가할 때 코드를 수정하지 않는다. JSON 설정 파일 하나로 동화책 전체를 정의한다:

하나의 config = {
    bookMeta: 제본 규격 (가로 250mm × 세로 200mm, 도련 3mm)
    illustrations: 일러스트 20장 (webp)
    versions: {
        A: 베이비 모드 (3~5세) — 의성어 중심 20페이지
        B: 키즈 모드 (6~9세) — 의미 설명 20페이지
    }
    각 페이지: 장면, 일러스트, 텍스트, 사진 배치 좌표
}

이 구조 덕분에 대표가 새 스토리를 설계하면, 일러스트 + config 파일만 넘기면 바로 프로덕션에 올라간다. 개발자가 코드를 건드릴 필요 없음.

⑤ 한국어 자연어 처리

아이 이름이 동화 텍스트에 자연스럽게 들어가야 한다. 한국어는 받침 유무에 따라 조사가 달라진다 ("도현이가" vs "하준이가", "도현이를" vs "하준이를"). 9가지 조사 변환을 자동 처리하여, 어떤 이름을 넣어도 문법적으로 자연스러운 동화가 생성된다.

왜 쉽게 복제할 수 없는가

  1. 6개 AI 모델의 조합: 단일 배경 제거 API를 쓰는 것과 차원이 다름. 사진 특성별 최적 모델 선택 로직은 수천 장의 아이 사진 처리 경험에서 나옴.
  2. 아이 사진 특화: 일반적인 배경 제거와 달리, 아이의 머리카락, 손에 든 장난감, 모자, 특이한 포즈 등 아동 사진 고유의 문제를 해결.
  3. 인쇄 품질 기준: 화면에서 괜찮아 보이는 것과 300DPI 인쇄에서 괜찮은 것은 다름. 인쇄 사고를 0%에 가깝게 만드는 검수 기준은 실제 인쇄 경험 없이는 만들 수 없음.
  4. 파이프라인 통합: 배경 제거 → 포즈 분석 → 스마트 크롭 → 품질 검수 → 장면 배치 → 미리보기 → 인쇄 파일 생성이 하나의 자동화된 체인으로 동작. 각각은 오픈소스로 가능하지만, 이 체인을 인쇄 품질 수준으로 안정화하는 데 시간이 걸림.

현재 구현된 기술적 보호 조치

전송 구간

  • 모든 사진 업로드/다운로드는 HTTPS(TLS 1.2+) 암호화
  • AI 서버 접근은 Cloudflare Tunnel 경유 (서버 IP 비노출, DDoS 방어 포함)
  • 고정 도메인(ai.monviestory.co.kr) 사용, 직접 포트 노출 없음

처리 구간

  • AI 처리 서버는 자체 보유 GPU 서버 (외부 클라우드가 아닌 물리 서버)
  • 사진은 처리 중 메모리에만 존재, 처리 완료 즉시 메모리 해제 (clear_gpu_memory())
  • 업로드 파일 검증: 허용 포맷(JPEG, PNG, WebP, HEIC)만 수용, 20MB 제한
  • CORS 정책으로 허가된 도메인에서만 API 호출 가능
  • 요청 로깅: 모든 API 호출을 IP, 경로, 응답코드와 함께 기록 (감사 추적)

저장 구간

  • 최종 인쇄 파일: Firebase/NAS에 암호화 저장
  • 배경 제거된 중간 파일: 인쇄 완료 후 삭제 대상
  • 고객 원본 사진: 서비스 완료 후 일정 기간 내 삭제

우리의 구조적 이점

대부분의 커스텀 동화책 업체는 고객 사진을 외부 클라우드 API(remove.bg 등)로 보내 처리한다. 우리는 자체 GPU 서버에서 처리하므로 아이 사진이 외부 서비스로 전송되지 않는다. 이것은 개인정보 관점에서 명확한 차별점이며, 부모에게 신뢰를 줄 수 있는 마케팅 포인트이기도 하다.

(참고: remove.bg API는 비표준 포맷 폴백용으로만 사용하며, 기본 처리 경로가 아님)
